# Images — production swap guide

The site ships with **labelled SVG placeholders** (navy/gold cards printed with
the filename + dimensions) so layouts are final before real photography exists.
This is how to swap them in without breaking anything, and the recommended size
for every slot.

See also [`performance.md`](./performance.md) for the `<Img>` component and
[design-system §7](../prompts/00-DESIGN-SYSTEM.md) for the canonical name list.

---

## How image paths resolve today

Components never hardcode paths. They call `placeholder('hero-campus')` or read
`IMAGES.heroCampus` from **`src/utils/assets.js`**, which currently resolves
every logical name to `/images/placeholders/<name>.svg` (generated by
`npm run gen:placeholders`).

So there is **one file to edit** for the production swap.

> **Brand logo — already swapped.** The real logo is live (hosted on
> Cloudinary) and centralised in `src/utils/assets.js` as `LOGO.normal` (light
> backgrounds) and `LOGO.white` (dark backgrounds), with `LOGO_SIZE` (441×74)
> for CLS-safe dimensions. The header swaps the variant by background (white
> over the hero, normal once solid), the footer uses the white variant, and the
> wide lockup includes the college name — so the separate wordmark text beside
> the logo was removed everywhere. `res.cloudinary.com` is preconnected in
> `index.html`. To change the logo, edit those two URLs in one place.

## Swapping in real photos

1. **Add the optimised files** under `public/images/` — keep the §7 kebab-case
   names, e.g. `public/images/hero-campus.jpg`, `public/images/course-bcom.webp`.
   Prefer **WebP/AVIF** (with a JPG kept for OG/social), compress to ~80 quality,
   and export at (roughly) the **2× of the display size** from the table below.
2. **Point the resolver at them** in `src/utils/assets.js`. Either flip the
   default extension/directory, or add a small per-name override map, e.g.:

   ```js
   // src/utils/assets.js — production override example
   const REAL = {
     'hero-campus': '/images/hero-campus.jpg',
     'course-bcom': '/images/course-bcom.webp',
     // …only the names you have real files for
   };
   export function placeholder(name) {
     const base = String(name).replace(/^.*\//, '').replace(/\.\w+$/, '');
     return REAL[base] || `/images/placeholders/${base}.svg`; // graceful fallback
   }
   ```

   Anything without a real file keeps showing its labelled placeholder, and
   `<Img>`'s `onError` fallback covers a mistyped path.
3. **Update the hero preload** in `public/index.html` to the real hero file and
   MIME type so the LCP keeps its high-priority head start:

   ```html
   <link rel="preload" as="image" href="%PUBLIC_URL%/images/hero-campus.jpg"
         type="image/jpeg" fetchpriority="high" />
   ```
4. **Pass real `width`/`height`** to `<Img>` where a slot doesn't already get
   its box from CSS `aspect-ratio`, so there is zero layout shift.
5. Rebuild (`npm run build`) and re-run Lighthouse (see `performance.md`).

> Tip: keep raster source files out of git history if they're large; commit the
> optimised, web-ready versions only.

## Recommended dimensions per slot

Intrinsic export sizes (the placeholder generator uses these). Ship ~2× for
crisp rendering on high-DPI phones, then let the CSS box scale them down.

| Slot (logical name) | Size (px) | Aspect | Used on |
| --- | --- | --- | --- |
| `logo-icon-commerce` | 360 × 96 | ~15:4 | header, footer, drawer, admin login |
| `favicon` | 256 × 256 | 1:1 | browser tab / PWA |
| `hero-campus`, `hero-students`, `hero-library` | 1600 × 900 | 16:9 | home + page heroes (LCP) |
| `about-college-building`, `about-campus-aerial`, `vision-mission` | 1200 × 800 | 3:2 | About / Vision sections |
| leadership portraits (`principal-dr-mandira-saha`, `president-dipali-bora`, `advisor-debasish-bora`, `rector-sawpon-dowerah`, `director-academic-rajib-das`, `director-dipanju-bora`, `academic-advisor-nilanjan-bhattacharjee`) | 600 × 750 | 4:5 | Leadership |
| `course-bcom`, `course-bba`, `course-bca`, `course-ba` | 800 × 600 | 4:3 | Courses cards/detail |
| `dept-arts`, `dept-commerce`, `dept-science` | 800 × 600 | 4:3 | Departments |
| `facility-library`, `facility-computer-lab`, `facility-canteen`, `facility-playground`, `facility-smart-classroom`, `facility-wifi` | 800 × 600 | 4:3 | Facilities |
| `faculty-placeholder` | 600 × 750 | 4:5 | Faculty cards |
| `gallery-1` … `gallery-12` | 800 × 600 | 4:3 | Gallery (masonry crops these) |
| `event-college-week`, `event-cooking-competition`, `event-icon-shield`, `event-icon-trophy` | 1200 × 800 | 3:2 | Events |
| `testimonial-avatar` | 400 × 400 | 1:1 | Testimonials |
| `prospectus-cover` | 800 × 1100 | ~3:4 | prospectus / admissions |
| `map-location` | 1200 × 600 | 2:1 | footer map thumbnail |
| `og-default` | 1200 × 630 | 1.91:1 | social share card (keep as JPG/PNG) |

If you need a new slot, add it to `scripts/gen-placeholders.mjs` and
`src/utils/assets.js` using the same kebab-case rule, and note it in the PR.
